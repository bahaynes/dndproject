## Commands

### Development Environment
```bash
./kube/dev.sh          # Start (builds images if needed)
./kube/teardown.sh     # Stop all containers
```
Frontend: http://localhost:5173 | Backend: http://localhost:8000

### Running Tests
```bash
./run_tests.sh                            # All tests (backend + frontend)
cd backend && uv run pytest tests/        # Backend only
cd backend && uv run pytest tests/test_main.py  # Single test file
cd frontend && pnpm test:ci               # Frontend only (non-interactive)
```

### Database Migrations
```bash
make migrate                    # Apply pending migrations (inside container)
make revision m="Add column x"  # Create new autogenerated migration
```

### Frontend
```bash
cd frontend && pnpm lint         # Prettier + ESLint
cd frontend && pnpm check        # Svelte type check
```

## Architecture

### Multi-Tenancy
Every user is scoped to a **Campaign** (`users.campaign_id`). JWT tokens encode both `discord_id` and `campaign_id`. All queries must filter by `campaign_id` — never return cross-campaign data.

Two auth dependency levels in `backend/app/dependencies.py`:
- `get_current_active_user` — any active campaign-scoped user
- `get_current_active_admin_user` — DM/admin only (`role == "admin"`)

### Backend Module Pattern
Each feature lives in `backend/app/modules/<name>/` with:
- `models.py` — SQLAlchemy ORM models
- `schemas.py` — Pydantic request/response schemas
- `service.py` — business logic / DB queries
- `router.py` — FastAPI routes

**When adding a new module**, you must import its models in **both**:
1. `backend/app/all_models.py` — so Alembic detects schema changes
2. `backend/app/main.py` — so SQLAlchemy creates tables and the router is registered

All routes are prefixed `/api/<name>` in `main.py`.

### Database
- Dev: SQLite (`sqlite:///./test.db`)
- Prod: PostgreSQL via psycopg 3 driver (`postgresql+psycopg://`)
- `database.py` auto-converts `postgres://` / `postgresql://` URLs
- Tests use an in-memory SQLite DB (`sqlite:///:memory:`) — see `backend/tests/conftest.py`
- SQLAlchemy JSON columns: mutations must replace the whole object, not mutate in-place

### Frontend Patterns
- API calls: use `API_BASE_URL` from `$lib/config` (resolves to `VITE_API_URL` env var or `/api`)
- Auth token: `get(auth).token` from `$lib/auth` store
- Shared types: `frontend/src/lib/types.ts`
- Package manager: **pnpm**; backend uses **uv**

### LLM / One-Shot Generator
`backend/app/llm_service.py` wraps an OpenRouter-compatible API. Configured via `LLM_API_BASE`, `LLM_API_KEY`, `LLM_MODEL` env vars. The oneshot module polls async generation jobs at `/api/oneshot/{id}`.

### Package Managers
- Backend: `uv` (lock file: `uv.lock`)
- Frontend: `pnpm` (lock file: `frontend/pnpm-lock.yaml`)
- Do not switch these without discussion.
